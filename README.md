# Руководство пользователя

## Автоматизированная система учёта заявок в IT-отдел (IT Help Desk)

---

## 1. Введение

Руководство пользователя предназначено для работы с программой «Автоматизированная система учёта заявок в IT-отдел (IT Help Desk)». Программа позволяет сотрудникам организации создавать заявки на техническую поддержку, а IT-специалистам — обрабатывать и решать эти заявки.

### Основные возможности программы:

- Регистрация и авторизация пользователей
- Создание заявок на техническую поддержку
- Просмотр и управление заявками
- Добавление комментариев и решений
- Оценка качества решения заявок
- Фильтрация и поиск заявок
- Статистика по заявкам

---

## 2. Установка и запуск программы

### 2.1. Требования к системе

- Операционная система: Windows 10 или 11 (64-разрядная)
- Установленный .NET Framework 4.7.2 или выше
- Доступ к базе данных Microsoft SQL Server

### 2.2. Установка программы

1. Скопируйте папку с программой на рабочий компьютер
2. Убедитесь, что файл конфигурации `App.config` содержит корректную строку подключения к базе данных

### 2.3. Запуск программы

1. Найдите исполняемый файл `ITSupport.exe`
2. Дважды кликните по нему для запуска
3. Откроется окно авторизации

---

## 3. Авторизация в системе

Для входа в систему необходимо выполнить следующие действия:

1. В поле **"Логин"** введите ваш логин (например, `ivanov`)
2. В поле **"Пароль"** введите ваш пароль (например, `123`)
3. Нажмите кнопку **"Войти"**

**Результат:**
- Если данные введены корректно, откроется форма сотрудника или IT-специалиста (в зависимости от вашей роли)
- Если данные введены неверно, появится сообщение **"Неверный логин или пароль"**

---

## 4. Регистрация нового пользователя

Если у вас ещё нет учётной записи, необходимо зарегистрироваться:

1. На форме авторизации нажмите кнопку **"Регистрация"**
2. Откроется форма регистрации
3. Заполните все поля:
   - **Логин** — придумайте уникальное имя пользователя (не менее 3 символов)
   - **Пароль** — придумайте надёжный пароль (не менее 4 символов)
   - **Подтверждение пароля** — введите пароль ещё раз
   - **ФИО** — ваши фамилия, имя и отчество
   - **Электронная почта** — ваш email (необязательно)
   - **Телефон** — ваш номер телефона (необязательно)
4. Нажмите кнопку **"Зарегистрироваться"**

**Результат:**
- При успешной регистрации появится сообщение **"Аккаунт создан"**
- Вы будете автоматически возвращены на форму авторизации
- Теперь вы можете войти в систему с созданными данными

---

## 5. Работа сотрудника

После успешной авторизации открывается форма сотрудника.

### 5.1. Просмотр списка заявок

На форме сотрудника отображаются только ваши заявки (созданные вами).

**Информация в таблице:**

| Колонка | Описание |
|---------|----------|
| № | Уникальный номер заявки |
| Проблема | Краткое описание проблемы |
| Категория | К какой категории относится проблема |
| Приоритет | Важность заявки |
| Статус | Текущий статус обработки |
| Создана | Дата и время создания |

**Обновление списка:** Нажмите кнопку **"Обновить"** для загрузки актуальных данных.

### 5.2. Поиск заявок

Вы можете найти нужную заявку по ключевым словам:

1. В поле **"Поиск"** введите ключевое слово (например, "принтер")
2. Список автоматически обновится, показывая только заявки с этим словом в описании

### 5.3. Создание новой заявки

Для создания новой заявки на техническую поддержку:

1. Нажмите кнопку **"Новая заявка"**
2. Откроется форма создания заявки
3. Заполните поля:
   - **Описание проблемы** — подробно опишите, что случилось (обязательное поле)
   - **Приоритет** — выберите важность проблемы (Высокий, Средний, Низкий)
   - **Категория** — выберите категорию проблемы (Оборудование, Программное обеспечение, Сеть и т.д.)
4. Нажмите кнопку **"Создать"**

### 5.4. Просмотр деталей заявки

Чтобы просмотреть подробную информацию о заявке:

1. Выберите заявку в списке (кликните по строке)
2. Нажмите кнопку **"Открыть"** или дважды кликните по строке
3. Откроется форма просмотра заявки

На этой форме вы можете увидеть:
- Описание проблемы
- Текущий статус
- Приоритет и категорию
- Дату создания
- Комментарии к заявке
- Решение (если заявка решена)
- Блок оценки (если заявка решена)

### 5.5. Добавление комментария

Вы можете добавить комментарий к заявке:

1. В поле **"Добавить комментарий"** введите текст
2. Нажмите кнопку **"Добавить комментарий"**

**Результат:** Комментарий появится в списке комментариев с указанием вашего имени и даты.

### 5.6. Оценка заявки

Когда IT-специалист решит вашу проблему (статус заявки изменится на "Решена"), вы сможете оценить качество решения:

1. Откройте заявку (см. раздел 5.4)
2. В блоке **"Оценка"** выберите значение от 1 до 5
3. Нажмите кнопку **"Сохранить оценку"**

**Результат:** Оценка сохранена, сообщение **"Спасибо за оценку!"**. Кнопка оценки становится недоступной.

### 5.7. Закрытие заявки

После того как вы оценили решение, вы можете закрыть заявку:

1. Откройте заявку (см. раздел 5.4)
2. Нажмите кнопку **"Закрыть заявку"**
3. Подтвердите действие, нажав **"Да"** в диалоговом окне

**Результат:** Статус заявки изменён на "Закрыта". Заявка считается полностью завершённой.

### 5.8. Выход из системы

Чтобы выйти из системы:

1. Нажмите кнопку **"Выход"** в правом верхнем углу
2. Форма закроется, вы вернётесь к форме авторизации

## 6. Работа IT-специалиста

После успешной авторизации открывается форма IT-специалиста.

### 6.1. Просмотр списка заявок

На форме IT-специалиста отображаются **все** заявки из системы.

**Информация в таблице:**

| Колонка | Описание |
|---------|----------|
| № | Уникальный номер заявки |
| Сотрудник | ФИО сотрудника, создавшего заявку |
| Проблема | Краткое описание проблемы |
| Категория | К какой категории относится проблема |
| Приоритет | Важность заявки |
| Статус | Текущий статус обработки |
| Создана | Дата и время создания |

**Обновление списка:** Нажмите кнопку **"Обновить"** для загрузки актуальных данных.

### 6.2. Фильтрация заявок

Вы можете отфильтровать заявки по статусу и приоритету:

**Фильтр по статусу:**
1. Выберите статус в выпадающем списке **"Статус"**
2. Доступные статусы: Все, Новая, В работе, Решена, Закрыта

**Фильтр по приоритету:**
1. Выберите приоритет в выпадающем списке **"Приоритет"**
2. Доступные приоритеты: Все, Высокий, Средний, Низкий

Список заявок обновится автоматически после выбора фильтра.

### 6.3. Поиск заявок

Вы можете найти заявку по ключевым словам:

1. В поле **"Поиск"** введите ключевое слово
2. Поиск выполняется по описанию проблемы и имени сотрудника

### 6.4. Взятие заявки в работу

Чтобы начать работу над новой заявкой:

1. Выберите заявку со статусом **"Новая"**
2. Нажмите кнопку **"Взять в работу"**

**Результат:** Статус заявки изменён на **"В работе"**. Вы можете добавить решение или изменить статус.

### 6.5. Просмотр деталей заявки

Чтобы просмотреть подробную информацию о заявке:

1. Выберите заявку в списке
2. Нажмите кнопку **"Открыть"** или дважды кликните по строке

На этой форме вы можете увидеть:
- Описание проблемы
- Текущий статус
- Приоритет и категорию
- Дату создания
- Комментарии к заявке (включая внутренние)
- Решение (если есть)
- Блок управления статусом

### 6.6. Добавление комментария

IT-специалист может добавлять как публичные, так и внутренние комментарии:

1. Введите текст комментария в поле **"Комментарий"**
2. Если комментарий предназначен только для IT-специалистов, установите флаг **"Внутренний"**
3. Нажмите кнопку **"Добавить комментарий"**

**Результат:** Комментарий добавлен. Внутренние комментарии видны только IT-специалистам.

### 6.7. Добавление решения

Когда проблема решена, добавьте решение:

1. В поле **"Решение"** введите текст решения
2. Нажмите кнопку **"Сохранить решение"**

**Результат:** Решение сохранено, статус заявки автоматически изменён на **"Решена"**.

### 6.8. Изменение статуса заявки

Вы можете изменить статус заявки вручную:

1. В выпадающем списке **"Статус"** выберите новое значение
2. Нажмите кнопку **"Сохранить статус"**

**Доступные статусы:**
- **Новая** — заявка только что создана
- **В работе** — заявка взята в работу
- **Решена** — проблема решена
- **Закрыта** — заявка закрыта сотрудником

### 6.9. Просмотр статистики

На форме IT-специалиста отображается статистика по заявкам:

**Информация:**
- **Новые** — количество заявок со статусом "Новая"
- **В работе** — количество заявок со статусом "В работе"
- **Решённые** — количество заявок со статусом "Решена"
- **Всего** — общее количество всех заявок

### 6.10. Выход из системы

Чтобы выйти из системы:

1. Нажмите кнопку **"Выход"** в правом верхнем углу
2. Форма закроется, вы вернётесь к форме авторизации

## 7. Справочная информация

### 7.1. Статусы заявок

| Статус | Описание | Цвет |
|--------|----------|------|
| Новая | Заявка создана, ожидает обработки | Синий |
| В работе | Заявка взята в работу IT-специалистом | Жёлтый |
| Решена | Проблема решена, ожидает оценки | Зелёный |
| Закрыта | Заявка закрыта сотрудником | Серый |

### 7.2. Приоритеты заявок

| Приоритет | Описание | Цвет |
|-----------|----------|------|
| Высокий | Критическая проблема, требуется срочное решение | Красный |
| Средний | Проблема средней важности | Жёлтый |
| Низкий | Некритичная проблема | Зелёный |

### 7.3. Категории заявок

| Категория | Описание | Примеры |
|-----------|----------|---------|
| Оборудование | Проблемы с компьютерной техникой | Принтер не работает, компьютер не включается |
| Программное обеспечение | Проблемы с программами | Не открывается Word, ошибка в 1С |
| Сеть | Проблемы с интернетом и сетью | Нет доступа в интернет, не работает почта |
| Другое | Прочие проблемы | Требуется помощь с настройкой |

## 8. Часто задаваемые вопросы (FAQ)

**Вопрос 1: Я забыл пароль. Что делать?**

**Ответ:** Обратитесь к IT-специалисту. Администратор может сбросить пароль или создать нового пользователя.

**Вопрос 2: Почему я не вижу заявку, которую только что создал?**

**Ответ:** Нажмите кнопку **"Обновить"** на форме сотрудника. Список заявок обновится, и новая заявка появится.

**Вопрос 3: Я не могу взять заявку в работу. Почему?**

**Ответ:** Заявку можно взять в работу только если её статус **"Новая"**. Если статус другой (например, "В работе"), кнопка не сработает.

**Вопрос 4: Я хочу оценить заявку, но у меня не получается.**

**Ответ:** Оценить можно только заявку со статусом **"Решена"**. Если заявка ещё не решена, блок оценки не отображается.

**Вопрос 5: Как удалить заявку?**

**Ответ:** Удаление заявок не предусмотрено. Все заявки остаются в системе для ведения истории.

**Вопрос 6: Сколько заявок я могу создать?**

**Ответ:** Количество заявок не ограничено.

**Вопрос 7: Как узнать, кто взял мою заявку в работу?**

**Ответ:** В карточке заявки можно посмотреть авторов комментариев. Если IT-специалист добавил комментарий или решение, вы увидите его имя.

## 9. Заключение

Программа «Автоматизированная система учёта заявок в IT-отдел» позволяет эффективно управлять процессом обработки заявок на техническую поддержку.

**Для сотрудников:**
- Быстрое создание заявок
- Прозрачность статуса обработки
- Возможность оценить качество решения

**Для IT-специалистов:**
- Удобное управление всеми заявками
- Фильтрация и поиск
- Возможность вести внутренние заметки
- Статистика для контроля нагрузки

---

*При возникновении вопросов или проблем обращайтесь к администратору системы.*
